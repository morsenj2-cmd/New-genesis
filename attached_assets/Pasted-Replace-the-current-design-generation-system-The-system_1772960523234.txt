## CORE PROBLEMS

1. The generator currently changes only surface properties such as:

colors
rounded corners
spacing
minor component styling

This means the underlying layout architecture remains identical.

2. Context detection currently relies on keyword detection instead of semantic understanding of the full prompt.

Because of this, the system often misinterprets user intent.

The generator must instead:

• interpret the **entire prompt semantically**
• derive context using the internal LLM
• enrich context using internet retrieval
• then generate structure based on that context

The generator must produce **entirely different page structures**, not style variations.
