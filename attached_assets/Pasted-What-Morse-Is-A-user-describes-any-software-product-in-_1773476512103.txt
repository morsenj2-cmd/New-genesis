What Morse Is
A user describes any software product in plain English. Morse generates a complete, runnable, uniquely designed application — frontend, backend, and all functionality — and exports it as a downloadable ZIP.

What Gets Exported (the ZIP the user downloads)
my-product/
├── index.html          ← loads fonts, mounts React
├── src/
│   ├── main.jsx        ← ReactDOM.createRoot entry
│   ├── App.jsx         ← GEMINI-WRITTEN — the entire product
│   └── globals.css     ← genome design tokens as CSS vars
├── vite.config.js
├── package.json
├── server.js           ← GEMINI-WRITTEN (if app needs backend)
└── README.md
The user runs npm install && npm run dev and has a working product.

When a User Edits a Project (apply-nl)
User types an edit like "make it darker" or "add a pricing section"
        │
        ▼
  geminiInterpretEdit()
  Gemini classifies: style_change | layout_modification |
                     content_change | regenerate | compound
        │
        ▼
  Result merged with local router (routePromptAsync)
  Patches applied to genome/layout/content as appropriate
        │
        ▼
  If shouldRegenerate = true → full pipeline reruns
  If style/layout change → genome or DNA patched and re-rendered

Fallback Behaviour
At every stage, if Gemini fails (no API key, network error, malformed response), the system catches the error and falls back to the existing local engines. The project still generates — just with generic content instead of Gemini-written content. Nothing breaks.
