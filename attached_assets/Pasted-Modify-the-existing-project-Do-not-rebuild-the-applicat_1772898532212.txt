Modify the existing project. Do not rebuild the application and do not change Clerk authentication or the Neon database configuration.

Goal

Replace the current keyword-based prompt and NL editing system with a semantic prompt interpreter that behaves like an LLM understanding user intent.

The system must:

• understand varied prompts
• understand editing instructions
• infer context
• modify the design state reliably

------------------------------------------------
1. UNIFIED PROMPT INTERPRETER
------------------------------------------------

Create a new module

/lib/semanticInterpreter.ts

Purpose

Convert natural language into structured intent.

Input

string prompt

Output

{
intent
target
property
value
confidence
}

Example

Prompt

"change the name from flowbase to morse"

Output

{
intent: "modify",
target: "brand.name",
value: "Morse"
}

Example

Prompt

"use a blue color scheme"

Output

{
intent: "modify",
target: "theme.primaryColor",
value: "blue"
}

Example

Prompt

"make this a cloud storage platform"

Output

{
intent: "set_product_type",
value: "cloud_storage"
}
